CASE 04 / 10
Alpha Group

The operations layer: billing, brokers, bookings, and HR

React/NodeLambdaEventBridgeSES/SNSMS Graph
The problem

Three very different businesses — insurance, hospitality, fitness — each running on manual processes that didn't scale: paper-heavy underwriting, phone-in reservations, and monthly billing done by hand.

What I architected, built & collaborated on

Working with the Alpha engineering team across all four businesses: for insurance, CRM systems for underwriting, claims, and broker management, customer-facing quote calculators and policy portals, a broker onboarding workflow, and REST APIs integrating underwriting partners, banks, and reinsurers. For the restaurant, a real-time booking and capacity management system. For the gym, a membership platform plus an automated recurring debit order run — scheduled batch billing with bank reconciliation, on Lambda, EventBridge, S3, and SES/SNS. Plus an internal HR portal with org-chart, employee self-service, and a wellness session booking system, integrated with Microsoft Graph.

Where it stands

All in production — policy processing time down 60%, manual data-entry errors down 85%.

CASE 05 / 10
Alpha Group

Killing a decade-old Adobe dependency, one form at a time

PDF ParsingE-SignatureNode.jsBrowser-Native
The problem

Hundreds of claims forms were trapped in a deprecated Adobe-only PDF format — customers on Chrome, Safari, or a phone hit a dead end and had to install desktop software just to file a claim.

What I architected & collaborated on

Working with the Alpha engineering team, helped architect and build an in-browser pipeline that reads each form's own underlying blueprint and reconstructs it as a fully interactive web form — conditional logic, calculations, and draw-to-sign included — then exports a pixel-accurate PDF on submission. Alongside it, an in-house document execution and e-signature platform replacing a paid third-party subscription entirely.

Where it stands

Rolled out across the full form library, working end-to-end from browse to signed, submitted PDF; the e-signature platform is live handling agreement workflows.

CASE 07 / 10
123tutors

Turning a 5-day manual review into a 2-hour automated one

KYC APIBiometricsAML ScreeningSupabase
The problem

Every tutor application was manually reviewed by an admin before going live — a human bottleneck that couldn't scale past roughly 50 applications a week, with no verifiable identity or background checks behind it.

What I built

An automated vetting pipeline integrating a South African identity verification API (InTouch) — government ID verification, facial-match and liveness biometrics, AML/sanctions screening, and bank account verification, running in parallel with a tiered auto-approve/flag-for-review decision engine.

Where it stands

Vetting time down from 2–5 business days to under 2 hours; admin time per application from ~30 minutes to ~2 minutes (review queue only); scalability ceiling removed entirely.

React NativeNFCPaystackMulti-app

CASE 10 / 10
Alpha Insure

Testing an actuarial pricing engine before it touches a real quote

React + ViteNode/ExpressFirebase
The problem

Alpha Insure's motor rating engine runs on Akur8, a third-party actuarial pricing API — but validating hundreds of pricing scenarios by hand before every deploy wasn't sustainable.

What I built

A full-stack UAT tool that bulk-runs CSV-defined test cases against the Akur8 rating API sequentially, tracks live pass/fail/error counts, flags invalid rows before they burn a test run, exports results for re-import, and keeps a searchable history of every batch in Firestore.

Where it stands

In active use for pre-deploy validation of Alpha Insure's motor pricing.
