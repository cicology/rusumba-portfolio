The project

One organisation, three completely different businesses under it: an insurance operation, a restaurant, and a gym. Each had its own applications, its own users, and its own login — but all of it had to be centrally managed, secure, and consistent.

The answer was a single authentication gateway every application could sit behind, rather than each one reinventing login on its own.

What I did

  • Architected the gateway and worked with the Alpha engineering team to build it — a centralised auth service every application in the ecosystem authenticates through.
  • Built on AWS Cognito with KMS-signed JWTs, so session tokens are verifiable and tamper-evident across services.
  • Implemented an OAuth2/PKCE flow on Lambda and API Gateway, with DynamoDB backing identity and session state.
  • Collaborated across the team on rollout to underwriting, client-facing, and internal HR systems without disrupting existing users.

Where it landed

In production, underpinning underwriting, client-facing, and internal HR systems across the group. One login layer instead of many — centrally managed, and secure by design.
